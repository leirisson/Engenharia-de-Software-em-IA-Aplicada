Projeto 2 — Programador Agente (Livro: O Programador Pragmático)

Objetivo:
Processar o livro em PDF, gerar embeddings localmente e permitir busca semântica no Neo4j (topK trechos relevantes).

Fonte de dados (PDF do projeto):
src/dados/O Programador Pragmático - Autor (Andrew Hunt e David Thomas).pdf

Checklist de tarefas

[ ] 1) Preparação do ambiente
- Rodar `npm ci`
- Criar `.env` com:
  - NEO4J_URI, NEO4J_USER, NEO4J_PASSWORD
  - EMBEDDING_MODEL
- Subir Neo4j: `npm run infra:up`

[ ] 2) Ajustar configurações do projeto
- Em `src/config.ts`, atualizar `pdf.path` para o caminho do PDF do livro
- Confirmar `neo4j.nodeLabel` (ex.: `Livro`) e `neo4j.indexName`
- Definir `chunkSize` e `chunkOverlap` iniciais (ex.: 1000/200)

[ ] 3) Implementar o processador do documento (processadorDeDocumentos.ts)
- Ler o PDF e extrair texto
- Gerar `Document[]` com metadados úteis:
  - `source` (arquivo)
  - `pageNumber` (se disponível)
- Aplicar `RecursiveCharacterTextSplitter` e retornar chunks

[ ] 4) Implementar embeddings locais
- Instanciar `HuggingFaceTransformersEmbeddings` com `EMBEDDING_MODEL`
- Fazer um teste mínimo: gerar embedding de uma string curta e validar que retorna um vetor

[ ] 5) Conectar no Neo4j Vector Store
- Criar a instância do `Neo4jVectorStore.fromExistingGraph(...)`
- Garantir que o índice vetorial existe/é reutilizado

[ ] 6) Ingestão de chunks no Neo4j
- Estratégia simples (primeira versão):
  - limpar o label `Livro` e inserir tudo novamente
- Estratégia recomendada (incremental/dedup):
  - gerar `chunkId` + `contentHash` por chunk
  - criar constraint UNIQUE em `chunkId`
  - fazer `MERGE` (upsert) para inserir só o que mudou

[ ] 7) Busca semântica (consulta)
- Criar uma lista fixa de 5 perguntas sobre o livro (conceito, prática, exemplos)
- Rodar `similaritySearch`/`similaritySearchWithScore` com topK configurável
- Exibir resultados no terminal usando `src/util.ts`

[ ] 8) Validação da qualidade
- Ajustar `chunkSize/chunkOverlap` se o topK vier “quebrado” ou irrelevante
- Validar se os resultados mostram trechos coerentes e (se houver) a página

[ ] 9) Fechamento
- Documentar como rodar:
  - `npm run infra:up`
  - `npm run dev`
  - `npm run infra:down`
